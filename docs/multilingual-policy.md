# 다국어 및 양방향 번역 정책 — PoomSchool(품스쿨)

## 1. 기본 원칙

PoomSchool(품스쿨)은 특정 4~5개 언어로 지원 범위를 제한하지 않습니다. AI Provider가 처리할
수 있는 다양한 언어와 한국어 사이의 양방향 번역을 제공합니다.

사용자에게는 다음과 같이 안내합니다.

> "입력 언어를 자동 감지하고, AI 모델이 지원하는 다양한 언어와 한국어 사이의 양방향 번역을
> 제공합니다. 번역 신뢰도가 낮은 경우 원문 확인이나 언어 선택을 요청합니다."

"전 세계 모든 언어를 완벽하게 지원한다"고 안내하지 않습니다.

## 2. 번역 방향

1. **학교 → 부모**: 학교가 작성한 한국어 안내문을 부모가 사용하는 언어로 번역하고, 쉬운
   한국어 설명을 함께 제공합니다. (`analyze_notice`, `translate_notice`)
2. **부모 → 학교**: 부모가 자신의 모국어로 입력한 내용을 담임교사에게 전달할 자연스럽고
   정중한 한국어로 변환하고, 부모가 의미를 확인할 수 있도록 역번역을 제공합니다.
   (`draft_teacher_message`)

## 3. 언어 식별 규칙

- 언어 식별에는 BCP-47 코드를 사용합니다 (예: `ko`, `vi`, `zh-CN`, `zh-TW`, `en`, `ru`,
  `th`, `tl`, `mn`, `km`, `uz`, `ne`, `si` 등).
- `source_language`는 BCP-47 코드 또는 `auto`를 허용합니다.
- `target_language`는 BCP-47 코드를 받으며, 고정된 enum으로 제한하지 않습니다.
- 사용자가 언어를 직접 지정하면 자동 감지 결과보다 우선합니다.

## 4. 언어 자동 감지 및 신뢰도

- `src/services/languageDetectionService.ts`가 문자 스크립트 기반 1차 감지를 수행합니다.
- 감지 신뢰도가 낮은 경우(기본 임계값 0.6 미만) 결과에 확인 요청 문구를 포함합니다.
- 감지가 불확실한 라틴 문자 전용 텍스트(en/vi/tl/uz 등)는 낮은 신뢰도로 표시되어 사용자
  확인을 유도합니다.

## 5. 원문 보존 규칙

번역 과정에서 다음 항목은 변경하지 않습니다.

- 날짜, 시간
- 금액
- 전화번호
- URL
- 아이 별칭, 학교명 등 고유명사
- 한국 학교의 고유 제도명(스쿨뱅킹, 방과후학교, 돌봄교실 등)은 한국어 원문을 함께 표시합니다.

`translationService.ts`는 AI 번역 결과에서 이러한 요소가 누락되었는지 검증하는 안전망을
제공하며, 누락 시 번역 신뢰도를 낮추고 "확인 필요" 문구를 추가합니다.

## 6. 설명 모드 (explanation_mode)

한국 학교제도 자체에 익숙하지 않은 사용자를 위해 언어 번역과 별개로 설명 모드를 제공합니다.

- `general`: 일반 학부모 설명
- `multicultural_parent`: 한국어와 한국 학교문화가 익숙하지 않은 부모
- `north_korean_parent`: 남한 학교제도와 행정 용어가 익숙하지 않은 부모

이 모드는 **사용자가 직접 선택**합니다. 언어, 표현, 입력 내용을 근거로 사용자가
북한이탈주민인지, 어떤 국적인지 임의로 추론하지 않으며, 가정 배경을 필수 개인정보로
저장하지 않습니다.

## 7. 번역 실패/불확실성 처리

- 번역할 수 없거나 품질이 낮을 가능성이 있으면 내용을 임의로 만들어내지 않습니다.
- 결과 메시지(특히 `draft_teacher_message`)는 자동 전송하지 않으며, 부모가 확인한 뒤
  직접 전송하도록 안내합니다.
- 역번역(back-translation)을 제공해 부모가 번역 의미를 스스로 확인할 수 있게 합니다.
